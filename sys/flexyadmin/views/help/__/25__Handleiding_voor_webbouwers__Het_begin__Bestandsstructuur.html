Bestanden en mappen in de root

Onderstaande mappen en bestanden zie je als FlexyAdmin in een lege map heb geïnstalleerd:

.htaccess - Niet aanpassen
htaccess.htaccess - Backup van .htaccess
temp.htaccess -  Hernoem deze tijdelijk naar .htaccess en de tijdelijke pagina index_temp.html is actief. Handig tijdens grote updates.
index_temp.html - Dit kun je gebruiken als een tijdelijke startpagina. Eventueel zelf aan te passen naar wens.
index.php - De start van alles. Niet aankomen.
readme.txt - Zeer globale info
robots.txt - Zorgt ervoor dat zoekrobots niet in alle mappen kijkt en heeft een verwijzing naar sitemap.xml
changelog.txt - Globale informatie over laatste wijzigingen van FlexyAdmin
todo.txt - Dit bestand kun je gebruiken als kladblok.
update.txt - Hierin staat sumier hoe je eventueel kunt updaten naar nieuwere versies van FlexyAdmin
sitemap.xml - Hierin staat de menustructuur van de site. Wordt automatisch aangepast. Zorg ervoor dat dit bestand beschrijfbaar is.
bulk_upload - Map waarin met FTP bestanden kunnen worden gezet die allemaal tegelijk aan de site moeten worden toegevoegd. Bijvoorbeeld honderden foto's tegelijk.
db - Map met demo databases en database updates voor FlexyAdmin. Gebruik deze map ook om je database backups in te bewaren.
site - Dit is de map waar je als bouwer veel mee te maken hebt. Zie hieronder.
sys - De map met FlexyAdmin. Niets aandoen! In geval van updates zul je meestal deze map moeten vervangen.
userguide - (URL's naar) handleidingen van de basiselementen van FlexyAdmin (CodeIgniter, jQuery (UI), tinyMCE en FlexyAdmin zelf)

De rode mappen en bestanden zijn niet nodig voor de online site. Pas nooit wat aan in de map sys!

Bestanden en mappen in Site

Hier zie je de bestanden en mappen in de map Site:

index.html - Om te voorkomen dat hier per ongeluk rechtstreeks in gekeken kan worden. Dit zul je in meerdere submappen tegenkomen. Gewoon laten staan.
controller.php - Hierin zul je als gevorderde webbouwer veel kunnen aanpassen. Hier wordt bepaald wat er gebeurt bij welke URL en worden eventuele modules en views geladen en aangeroepen.
assets - In deze map komen alle bestanden die de site verder nodig heeft en die de browser moet kunnen benaderen. Zoals foto's, downloads, maar ook javascript bestanden.
cache - Map met cachebestanden als je caching gebruikt voor je site.
config - Hierin vind je config bestanden, bijvoorbeeld voor de database.
Enkelen hebben een _local suffix. Deze worden gebruikt voor lokale instellingen. Zo kun je eenvoudig een scheiding maken tussen je lokale en online instellingen.
Je vindt hier ook eventuele config bestanden voor modules en noodzakelijke config bestanden voor plugins.
helpers - Hier kun je eigen helpers in plaatsen voor ondersteuning van je eigen modules of plugins. Hier lees je meer over CodeIgniter helpers.
languages - Hierin vind je language bestanden gegroepeerd per taal. Standaard vind je hier de talen nl (Nederlands) en en (Engels). Je kunt zelf talen en taalbestanden toevoegen voor je modules. Of de bestaande taalbestanden aanpassen.
libraries - Map met eigen libraries (classes). Hierin vind je modules en plugins. Deze kun je zelf aanpassen of nieuwe toevoegen. Er zitten al een aantal voorbeeld modules in.
models - Map met CodeIgniter models voor je site. Lees hier meer over models.
stats - Map met xml bestanden voor de statistieken van de site.
views - De map met de views: de eigenlijke HTML bestanden met PHP code waar de data komt. Zelf aan te maken en aan te passen.
De belangrijkste is site.php, dit is de view waarin alles samen komt. Verder zijn standaard page.php (een pagina), links.php (voor de links module) en error.php (foutmelding voor als een pagina niet gevonden kon worden)

Bestanden en mappen in Site/Assets

Hier zie je de bestanden en mappen in de map Assets:

css - Plaats hier je stylesheets, en zorg voor een goede link vanuit de view site.php.
Standaard staan hier: text.css, layout.css, ie*.css en admin.css. Zie de comments van elke stylesheets.
img - Plaats hier de afbeeldingen die nodig zijn voor de styling. Dus geen afbeeldingen die de gebruiker moet kunnen uploaden.
js - Plaats hier javascript bestanden. Standaard staat er al een site.js (begin van je js code) en een jQuery template. Zorg net als bij stylesheets dat je bestanden worden geladen vanaf de site.php view.
lists - FlexyAdmin gebruikt deze map om lijsten in te plaatsen die gebruikt worden door tinyMCE. Niet aankomen of zelf aanpassen.
pictures - Dit is de standaard map waar gebruikers hun Foto's in kunnen uploaden. Je kunt zelf meerdere mappen aanmaken voor diverse soorten afbeeldingen.
downloads - Dit is de standaard map waar gebruikers hun downloadbare bestanden in kunnen uploaden.
_thumbcache - Deze map gebruikt FlexyAdmin voor het opslaan van thumbnails van alle afbeeldingen. Hoef je niets zelf aan te doen.
