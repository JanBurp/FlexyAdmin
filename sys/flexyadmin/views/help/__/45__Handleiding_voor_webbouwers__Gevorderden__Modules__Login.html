De login module kan gebruikt worden om bezoekers te laten inloggen op de site en aan de hand daarvan extra content te laten zien of een deel van de site afgeschermd te maken.

Het is aan te raden om deze module automatisch in te laden en enkele losse pagina's te maken voor logout en register/forgot_password.
Op die manier ben je er zeker van dat de module voor elke pagina werkt!

Instellingen

Een CSS class die aan de body tag wordt toegevoegd als een user is ingelogd:

$config['class']='user_logged_in'; 

De user_group (laten staan als je niet weer waarom je dit zou aanpassen):

$config['group_id']=4;// 4 is visitors which can't login to the backend 

Uri's van de diverse onderdelen (login/register/forgot_password). Je kunt de module de uri's ook zelf laten zoeken ('auto_uris'):

$config['auto_uris']=true;
// $config['login_uri']='';
// $config['register_uri']='';
// $config['forgotten_password_uri']=''; 

Views

De loginmodule gebruikt enkele views die te vinden zijn in site/views/login. Ook zijn daar per taal een drietal e-mail templates te vinden die de module gebruikt.

 

Methods

 

login

login.login

login.logout

login.register

LET OP: deze method werkt alleen correct met de volgende instelling in site/config/config.php:

$config['query_urls'] = TRUE;

LET OP: deze method werkt alleen correct met de volgende instelling in site/config/config.php:
